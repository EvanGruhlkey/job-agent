# Resume Job Agent

Resume Job Agent is a local CLI for discovering public job listings and saving them to readable Markdown or text files. It searches publicly available pages and posting endpoints, ranks matching roles, updates a local job index, and writes the results to `data/jobs.md` by default.

There is no web UI or local server.

## Features

- Search jobs by title, location, and desired result count from the terminal.
- Save discovered jobs to Markdown or plain text reports.
- Keep a structured local index in `data/job-index.json`.
- Re-list indexed jobs without running another web search.
- Tailor a resume for an indexed job and save the result as Markdown.

## Quick Start

Install dependencies:

```bash
npm install
```

Search for jobs and save a Markdown report:

```bash
npm start -- search --title "Software Engineer" --location "Chicago" --max 25
```

The command updates `data/job-index.json` and writes `data/jobs.md`.

Save plain text instead:

```bash
npm start -- search --title "Software Engineer" --location "Remote" --max 25 --output data/jobs.txt
```

List jobs already in the local index:

```bash
npm start -- list --query "Software Engineer" --output data/jobs.md
```

Tailor a resume to an indexed job:

```bash
npm start -- tailor --job <job-id-or-url> --resume ./resume.pdf --output data/tailored-resume.md
```

## Commands

| Command | Purpose |
| --- | --- |
| `npm start -- search` | Discover jobs, update the local index, and save a job report. |
| `npm start -- list` | Save matching jobs from `data/job-index.json` without searching the web. |
| `npm start -- tailor` | Generate a tailored resume Markdown file for an indexed job. |
| `npm start -- sources` | Print supported public source categories. |
| `npm start -- status` | Show CLI and LLM status. |

Common options:

| Option | Description |
| --- | --- |
| `--title`, `-t` | Target title for discovery. |
| `--query`, `-q` | Query for indexed jobs. |
| `--location`, `-l` | Location filter. |
| `--max`, `-n` | Number of jobs to request. |
| `--output`, `-o` | Output path. Defaults to `data/jobs.md`. |
| `--format` | `markdown` or `text`. The CLI also infers this from `.md` or `.txt`. |

## How It Works

The CLI discovers candidate listings, filters them against the search intent, ranks the best matches, saves structured job data locally, and writes a readable report.

| Discovery method | Description |
| --- | --- |
| Direct search | Opens supported job board search pages and parses visible job cards. |
| ATS discovery | Uses public Greenhouse, Ashby, and Lever board/posting endpoints when available. |
| Public web discovery | Searches public web results scoped to job-site domains, then fetches matching posting pages. |

## Project Structure

| Path | Purpose |
| --- | --- |
| `src/cli.js` | CLI entry point and report writer. |
| `src/jobIndex/` | Job discovery, indexing, source adapters, and matching logic. |
| `src/resumeParser.js` | Resume file/text extraction. |
| `src/tailor.js` | Deterministic resume tailoring fallback. |
| `data/job-index.json` | Local job index generated by discovery runs. |
| `data/jobs.md` | Default human-readable job report output. |
| `docs/` | Research and supporting notes. |

## Notes

This project is designed for local use and public-data discovery. It does not bypass login walls, CAPTCHAs, paywalls, or access controls. Search reliability depends on the availability and structure of public listing pages, which can change over time.
